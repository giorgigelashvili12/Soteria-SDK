# Version 1.1.0 - არსებული ფუნქციების განახლება და გაუმჯობესება

დამატებითი ფუნქცია და არსებული მეთოდების გაუმჯობესება სიჩქარის, გამოყენების და ოპტიმიზაციის მხრივ

## Node

**configure (NEW)** - იყენებს ორ არგუმენტს: passkey & secretKey. secretKey საჭიროა უსაფრთხოდ მონაცემების გადაწოდებისთვის. არ ქმნის სესიას, ახდენს ლოკალურ კონფიგურაციას არსებულ კლასში.

**createCheckout (NEW)** - createIntent-ის სანდო ალტერნატივა. საჭიროებს crypto მოდულს. თუ პროდუქტის ყიდვას ვუზრუნველყოფ და კლიენტიდან ვაგზავნი არსებულ მონაცემებს, მაშინ შესაძლებელია შეიცვალოს არსებული ველები. ძველი ფუნქციის გამოყენებით, ბაზასთან შედარება არსებული მონაცემების არ იქნებოდა ოპტიმიზაციისთვის ოპტიმალური. თუ ეს კონკრეტული, მიბმული "ხელმოწერა" შეიცვალა მონაცემების ცვლილებით, მაშინ მას სერვერი არანაირად მიიღებს.

**syncCatalog (NEW)** - ძველი ლოგიკით, პროდუქტის დამატება გიწევდა ხელით, და ეგეც შესაძლებელი იყო ერთ ინდივიდუალურ პროდუქტზე. ამ ფუნქციის გამოყენებით, შეგიძლიათ მონაცემების ინექცია, ათობით, ასობით თუ ათასობით. იყენებს MongoDB–ის BulkWrite-ს და ალგორითმს, რომელშიც: თუ პროდუქტი სიაში არსებობს, გამოტოვე და შემდეგზე გადადი. თუ არ არსებობს, მაშინ დაამატე.

# Version 1.2.0 - დამატებიტი ფუნქციები "მოქნილობისთვის"

სამი დამატებითი ფუნქცია, რომელიც მუშაობს პროდუქტებზე პლატფორმის გამოყენების გარეშე

**deleteProduct** - იღებს არსებული პროდუქტის ID-ს და მარტივად შლის სიიდან.

**editProduct** - იღებს არსებული პროდუქტის ობიექტს (საჭიროა მითითებული იყოს ID), და ანახლებს მასში მითითებულ ველებს.

**getProduct** - იღებს პროდუქტის ID-ს. თუ არ არის მითითებული, აბრუნებს პირველ პროდუქტს სიაში. თუ არის, მაშინ იპოვის და მოთხოვნილ პროდუქტს დააბრუნებს.
